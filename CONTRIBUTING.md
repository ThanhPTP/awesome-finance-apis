# Contributing to Awesome Finance APIs

Thanks for your interest in contributing! This list is maintained with a high quality bar — every entry should save a developer real time.

## What makes a good contribution

Before opening a PR, ask yourself:

- **Is this API genuinely useful?** It should solve a real problem for fintech devs, quants, or data scientists — not just exist.
- **Does it have a usable free or freemium tier?** "Contact sales for pricing" doesn't count. A developer should be able to sign up and make API calls within minutes.
- **Is the information accurate?** Have you personally verified the docs link works, the free tier exists, and the limitations are correct?
- **Is the description factual?** Write like an engineer, not a marketer. No superlatives, no hype.

## How to propose a new API

1. **Search first.** Check the README, open issues, and open PRs to make sure it isn't already listed or under review.
2. **One API per PR.** This keeps reviews fast and focused.
3. **Use the entry template below** so the list stays consistent.
4. **Place it alphabetically** within the correct category.

## Entry template

Copy this exactly and fill in the fields:

```markdown
- [**API Name**](https://link-to-docs-or-homepage) — One-sentence description of what the API provides.
  - **Best for:** Primary use case (e.g., "retail equity backtesting").
  - **Free tier:** Brief summary of limits (e.g., "500 req/day, 5 yrs history").
  - **Limitations:** Notable constraints (e.g., "delayed quotes only", "non-commercial license").
```

### Field guidelines

| Field | Required | Notes |
|---|---|---|
| Name + link | Yes | Link to official docs or homepage. No affiliate links, no tracking parameters. |
| Description | Yes | One sentence. Factual. No marketing copy or superlatives. |
| Best for | Yes | Who benefits most? Be specific (e.g., "daily equity backtesting", not "everyone"). |
| Free tier | Yes | Concrete limits. Approximate is fine if the provider doesn't publish exact numbers. |
| Limitations | Yes | Constraints a developer should know before signing up. |

## Other ways to contribute

- **Fix a broken link** — Open a PR or issue.
- **Correct inaccurate information** — If an API has changed its free tier, let us know.
- **Improve a description** — Clarity and accuracy are always welcome.
- **Suggest a new category** — Open an issue to discuss before adding.
- **Report a defunct API** — If a listed API has shut down or removed its free tier, open an issue.

## Quality guidelines

- Keep entries factual and concise.
- Do not add APIs that require paid plans with no usable free/freemium tier.
- Do not add APIs that are defunct, permanently down, or abandoned (no updates in 2+ years with known issues).
- Do not add wrappers, aggregators, or "API marketplaces" unless they provide genuine standalone value.
- Sort entries alphabetically within each category.
- Link to official documentation or homepage — not blog posts, affiliate pages, or landing pages with tracking parameters.

## Self-promotion policy

If you are affiliated with the API you're submitting (employee, founder, contractor, investor), you **must** disclose this in your PR description. Undisclosed self-promotion will result in the PR being closed and the contributor blocked.

Disclosed self-promotion is fine — we just need transparency. The entry will still be evaluated on the same quality criteria as everything else.

## Low-quality and spam contributions

The following will be **closed immediately without discussion**:

- **Link dumps** — PRs that add multiple unvetted APIs with minimal descriptions.
- **SEO spam** — Entries that exist primarily to generate backlinks rather than help developers.
- **Promotional copy** — Descriptions that read like marketing material or press releases.
- **Irrelevant tools** — APIs that aren't related to financial data (e.g., general-purpose CRMs, email tools, etc.).
- **Unverified AI-generated content** — Entries that appear to be generated by AI without human verification of accuracy (wrong URLs, hallucinated free tiers, nonexistent APIs).
- **Duplicate submissions** — APIs already listed or already proposed in open issues/PRs.
- **Entries with no usable free tier** — If you can't use the API without a credit card or sales call, it doesn't belong here.

Repeat offenders will be blocked from the repository.

## Maintainer rights

Maintainers reserve the right to close, edit, or decline any issue or PR without detailed justification. This isn't personal — it's how we keep the list high-quality with limited maintainer time. If your PR is closed, you're welcome to open a new one that addresses any feedback provided.

## Code of Conduct

All contributors are expected to follow our [Code of Conduct](CODE_OF_CONDUCT.md). The short version: be respectful, be constructive, don't spam.
